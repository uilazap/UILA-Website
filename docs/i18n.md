# Language detection & geofencing

The site is English (`en`) and Spanish (`es`). All strings are keyed in
`assets/js/i18n.js`. The flag toggle in the header lets visitors switch
manually; the logic below decides what loads on first paint.

## Detection priority (on first paint)

On every page load, `detectInitial()` in `assets/js/i18n.js` picks a
language in this order, stopping at the first match:

1. **URL override** ŌĆö `?lang=es` query string or `#es` hash fragment
   - Highest priority. Lets us share direct links.
   - Shareable URLs:
     - `https://www.uila.io/?lang=es` ŌåÆ Spanish
     - `https://www.uila.io/?lang=en` ŌåÆ English
     - `https://www.uila.io/#es` (alt form)
2. **`localStorage['uila.lang']`** ŌĆö the visitor's previous explicit
   choice on this device. Written only when the user clicks the flag
   toggle.
3. **`sessionStorage['uila.geoCountry']`** ŌĆö cached IP-country result
   from an earlier hit in the same session. Avoids re-querying the geo
   endpoint on every page load.
4. **`navigator.language`** ŌĆö browser-reported locale. `es-*` maps to
   `es`, everything else maps to `en`.

## Async geo hint (after paint)

After the initial synchronous `apply(detectInitial())`, we fire
`geoSwapIfUnset()`:

- **Skips entirely** if step 1 or 2 above already decided the language
  (URL override or prior explicit choice).
- Calls `https://api.country.is/` ŌĆö free, CORS-enabled, no API key,
  returns only `{country, ip}`.
- Caches the ISO 3166-1 alpha-2 country code in `sessionStorage` so
  later page loads in the same session don't re-query.
- If the code is in `SPANISH_COUNTRIES`, swaps the page to `es`;
  otherwise leaves it alone.

`SPANISH_COUNTRIES` currently contains:

```
AR ┬Ę BO ┬Ę CL ┬Ę CO ┬Ę CR ┬Ę CU ┬Ę DO ┬Ę EC ┬Ę SV ┬Ę GQ ┬Ę
GT ┬Ę HN ┬Ę MX ┬Ę NI ┬Ę PA ┬Ę PY ┬Ę PE ┬Ę PR ┬Ę ES ┬Ę UY ┬Ę VE
```

Geo hints **never write `localStorage`**. Only the user's flag click
does. That way a visitor who actively toggled EN while traveling
through SV stays on EN on their next visit.

## Trade-offs

- **Brief flash on first visit in a session.** Visitors see a quick
  render of whatever `navigator.language` chose (~100ŌĆō500 ms) before
  the geo call resolves and (maybe) swaps. Cached in `sessionStorage`
  after that, so only one flash per browser session.
- **Third-party IP exposure.** `api.country.is` receives the visitor's
  IP. For a public marketing site served from GitHub Pages this is
  comparable to the existing privacy surface (GitHub Pages already
  sees the IP), but it is one additional third party in the chain.
  If that isn't acceptable, remove the `geoSwapIfUnset()` call in
  `assets/js/i18n.js` ŌĆö the URL override + `localStorage` +
  `navigator.language` chain keeps working without it.
- **No server-side detection.** The site is a static GitHub Pages
  deploy, so there's no backend to inspect request headers. All
  detection runs in the browser after the HTML has already shipped.

## Keys and storage

| Key                        | Storage        | Written by           | Purpose                             |
|----------------------------|----------------|----------------------|-------------------------------------|
| `uila.lang`                | localStorage   | flag toggle / apply  | remembered explicit language choice |
| `uila.geoCountry`          | sessionStorage | geo lookup           | cached ISO country code             |

## Relevant code

- `assets/js/i18n.js` ŌĆö the whole detection + apply pipeline
- `index.html` ŌĆö every translatable element carries a `data-i18n="key"`
  attribute and a sensible English fallback as its initial text
